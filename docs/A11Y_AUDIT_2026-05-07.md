## HIGH Priority (Accessibility Blockers)

### 1. Icon-only buttons lack `aria-label` in Sim v2 palette
**File**: `src/features/simulation/SimulationViewV2.tsx:446`  
**Issue**: Palette buttons (Q1–Q65) render numbers with no accessible label for unanswered/flagged/answered state.  
**Fix**: Add `aria-label={`Question ${i + 1}${isFlagged ? ' flagged' : ''}${answered ? ' answered' : ''}`}` to button element (title already exists; expose to screen readers).

### 2. Ordering question list items receive click handlers without proper focus/keyboard semantics
**File**: `src/components/QuestionPlayer.tsx:198–216`  
**Issue**: `<li>` with `onClick` to set cursor is keyboard-inaccessible for users not using mouse; arrows (↑↓) buttons have `aria-label` but parent list item lacks role.  
**Fix**: Add `role="button"` and `tabindex="0"` to the `<li>`, or convert to `<button>` wrapper with semantic nesting; ensure keyboard event handlers for both J/K and arrow keys (already in place in `onKey` but li click needs keyboard support).

## MEDIUM Priority (Rough Edges)

### 3. Color-only signaling in Sim v2 palette + result bands
**File**: `src/features/simulation/SimulationViewV2.tsx:434–440` and `460–464`  
**Issue**: Palette buttons use `bg-ok/15` (green) for answered, `bg-warn/15` (amber) for flagged, and result bands (green/yellow/red) show readiness solely by color without additional text/icon indication.  
**Fix**: Palette already includes `title` tooltip; result bands have text labels. Add checkmark icon or status emoji (✓/⚠/✗) to palette buttons for visual clarity.

### 4. No `aria-live` for dynamic quiz feedback and timer countdown
**File**: `src/components/QuestionPlayer.tsx` (verdict announcement) and `src/features/simulation/SimulationViewV2.tsx` (countdown)  
**Issue**: Verdict (correct/incorrect/partial) appears after submit but is not announced to screen readers; final-minute toast uses `aria-live="polite"` (good) but timer itself is not live-region'd.  
**Fix**: Wrap verdict div with `role="status" aria-live="polite" aria-atomic="true"` in Verdict component; timer is visual-only (acceptable if not exam-mandatory).

### 5. Settings view checkbox labels lack explicit `for` or use implicit wrapping inconsistently
**File**: `src/features/settings/SettingsView.tsx:142–165`  
**Issue**: Checkboxes for "Reduce motion", "Beep on final minute", and "Exam-day focus mode" use implicit label wrapping (`<label><input>text</label>`), which is valid but mixed with explicit `htmlFor` labels elsewhere (lines 118, 130).  
**Fix**: Standardize all labels to explicit `htmlFor` pattern for consistency and clarity to assistive tech.

## LOW Priority (Paper-Cuts)

### 6. Focus outline precision could be tighter on small buttons
**File**: `src/styles/globals.css:47–50`  
**Issue**: Global `:focus-visible` outline is 2px with 2px offset; on `btn h-8 px-2 py-1` (move-up/down arrows), outline may visually overflow padding.  
**Fix**: No action required (global default is reasonable), but consider testing on 44×44 touch targets for adequate padding.

### 7. Keyboard shortcut legend missing Enter context for confidence + flag
**File**: `src/components/KeyboardShortcutsOverlay.tsx:19–29`  
**Issue**: Overlay lists S/U/G for confidence, F for flag, but does not mention space bar as alternative to Enter for submit (line 118 in QuestionPlayer checks `e.key === ' '`).  
**Fix**: Add row: `{ keys: 'Space', context: 'Question player', action: 'Submit / next (alternative to Enter)' }`.

### 8. Badge component (difficulty D1–D4, domain tags) uses color only in rare edge case
**File**: `src/components/QuestionPlayer.tsx:174–179` and `src/styles/globals.css:81–86`  
**Issue**: Badges display text (Domain, Subtopic, D{difficulty}) so no color-only signaling; badge variants (badge-ok, badge-warn, badge-bad) include text. Low-risk.  
**Fix**: None required; design already includes text labels.

### 9. Mobile bottom bar nav uses text abbreviations only (no icons)
**File**: `src/components/Layout.tsx:126–145`  
**Issue**: Mobile nav shows "Home", "Quiz", "Sim", "Sim2" (short text), then "More" menu. No icons may be harder for visual scanning on small screens.  
**Fix**: Consider adding icons (home, help-circle, zap, activity) above text for faster recognition; not a blocker but would improve UX.

## Mobile-Specific Findings

### 10. Touch target sizes on move-order arrows meet minimum
**File**: `src/components/QuestionPlayer.tsx:212–213`  
**Issue**: Buttons with `h-8 px-2 py-1` = 32px height × ~40px width; below WCAG 44×44px recommendation for touch.  
**Fix**: Increase to `h-10 px-3` or add horizontal spacing between buttons to avoid mis-taps in mobile-first view.

### 11. Sim v2 palette grid responsive but tight on mobile
**File**: `src/features/simulation/SimulationViewV2.tsx:428`  
**Issue**: `grid-cols-8 sm:grid-cols-10 md:grid-cols-[repeat(15,minmax(0,1fr))] lg:grid-cols-[repeat(20,minmax(0,1fr))]` yields 8-column grid on mobile with `h-10` buttons; acceptable but cramped on narrow viewports (< 320px).  
**Fix**: No fix required; responsive grid is appropriate. Test on iPhone SE (375px) to confirm readability.
